# Humanize Academic (KR) — 학술 논문 휴머나이저

AI가 생성한 한국어 학술 텍스트를 **사람이 쓴 논문처럼** 되돌린다. 핵심 철학은
"제거가 아니라 **재조정(rebalance)**"이다 — 범용 휴머나이저는 헤징·명사화·형식구조를
모두 'AI 티'로 보고 깎지만, **학술 글에서는 그것들이 규범**이다. 이 하네스는 AI 슬롭과
정당한 학술 규범의 경계선을 정밀하게 그어, 슬롭만 도려낸다.

## 0. 모드 결정 (Phase 0)

먼저 두 축을 확정한다.

### 글 무게 모드
- **`--academic` (기본값):** 논문·학위논문·초록·학술지 원고. 학술 규범 **보존**.
- **`--general`:** 에세이·블로그·보고서. 헤징·명사화·형식구조까지 공격적으로 다듬음
  (범용 휴머나이저 동작). 사용자가 '논문 아님'을 명시하거나 장르가 비학술일 때.

판단 규칙: 입력에 인용·각주·"본 연구/본고", IMRaD 섹션명, '~다' 평서체 학술 문체가
보이면 `--academic`. 모호하면 사용자에게 1회 확인.

### 처리 강도 모드
- **Fast (기본, ≤5,000자):** `academic-humanize-monolith` 단일 에이전트가
  탐지→교정→자가검증을 한 번에. 목표 2~3분.
- **Strict (≥8,000자 또는 `--strict`):** 5단계 파이프라인. 탐지·교정·충실도감사·자연성리뷰
  분리, 최대 3 교정 루프.

워크스페이스: `_workspace/{YYYY-MM-DD-NNN}/` (glob 시퀀싱). 입력은 `01_input.txt`에 보존.

## 1. 4대 불변 원칙 (절대 위반 금지)

1. **의미 불변 (신성불가침):** 사실·주장·수치·통계·고유명사·전문용어·직접인용·수식(equation)·
   참고문헌은 **한 글자도** 바꾸지 않는다. 위반 즉시 롤백.
2. **인용 날조 절대 금지:** 없는 인용을 만들지 않는다. 근거 없는 일반화는 *교정*이 아니라
   **플래그**(`[저자 확인: 출처 필요]`)로만 표시. AI 휴머나이저가 가짜 출처를 지어내면 학술
   부정이다 — 이 선을 넘지 않는다.
3. **학술 규범 보존:** 정당한 단일 헤징, 분야 표준 명사화·용어, 객관성 피동, IMRaD 구조,
   격식체는 **건드리지 않는다**. (근거: `references/academic-norms-keep.md`)
4. **근거 기반 + 과교정 방지:** 탐지된 span만 수정. 변경률 30% 경고, 50% 중단.
   탐지 안 된 문장은 그대로 둔다.

## 2. 탐지 대상 — Academic AI-Tell Taxonomy

전체 규칙은 `references/academic-tell-taxonomy.md` (11개 범주, 양면 범주 포함).
요약:

| # | 범주 | 심각도 | 모드별 |
|---|------|--------|--------|
| 1 | 공허한 메타·상투구 ("주목할 만한 점은", "시사하는 바가 크다") | S1 | 둘 다 |
| 2 | 근거 없는 일반화·단정 ("~로 알려져 있다"·인용 없는 "많은 연구") | S1 | 둘 다 → **플래그** |
| 3 | 환각·모호 인용 | S1 | 둘 다 → **플래그만** |
| 4 | Hedging 오용 (4a 중첩=교정 / 4b 단일=보존) | S2/KEEP | academic은 4b 보존 |
| 5 | 명사화·추상화 (5a 행위자은폐 연쇄=해체 / 5b 표준어=보존) | S2/KEEP | academic은 5b 보존 |
| 6 | 번역투 (~를 통해·~에 있어서·~되어진다·관계절 좌향수식·이중조사) | S1/S2 | 둘 다 |
| 7 | 리듬·구조 균일성 (균일 문장길이·drumbeat·단락 3~5문장 고정) | S2/S3 | 둘 다 |
| 8 | 접속사·전이 공식 (문두 "또한·게다가·나아가" 남발) | S2 | 둘 다 |
| 9 | Hype·주관 형용사 ("혁신적·놀랍게도·매우 중요한") | S1 | 둘 다 |
| 10 | 시각·서식 장식 (em-dash 남용·과볼드·이모지·산문 속 불릿) | S2 | 둘 다 |
| 11 | 피동 남용 (11a 기계적=일부 능동화 / 11b 객관성=보존) | S2/KEEP | academic은 11b 관대 |

**심각도:** S1=한 번만 나와도 AI 확정 → 처리. S2=3회+ 누적 시 처리. S3=다른 패턴과
중첩 시 강화. **`--academic`에서 4b·5b·11b는 절대 건드리지 않는다(보존 화이트리스트).**

## 3. 실행 흐름

### Fast 경로
1. Phase 0: 모드·런ID 확정, `01_input.txt` 저장.
2. `academic-humanize-monolith` 에이전트 1회 호출 → 탐지+교정+6점 자가검증.
3. `final.md` 출력 (교정문 + HTML 주석 메트릭 블록).

### Strict 경로
1. `academic-tell-detector` → `02_detection.json` (category·severity·span·fix·is_flag_only).
2. `academic-style-rewriter` → `03_rewrite.md` (탐지 span만 외과적 수정, 플래그는 주석).
3. 병렬 감사:
   - `scholarly-fidelity-auditor` → `04_fidelity.json` (의미·수치·인용 보존 13점 + 인용 날조 검사).
   - `academic-naturalness-reviewer` → `05_naturalness.json` (잔여 AI 티 + 과교정/규범훼손 검사).
4. 분기: fidelity 위반 OR 과교정 → rewriter 재호출(최대 3루프). naturalness 미달 → 재탐지.
5. `final.md` + 감사 JSON.

## 4. 출력 형식

`final.md` 끝에 HTML 주석으로 메트릭:
```
<!--
mode: academic | general
탐지: {범주별 카운트}
변경률: NN%
보존 화이트리스트 적용: {헤징/명사화/피동 보존 건수}
플래그(저자 확인 필요): {인용/근거 필요 건수}
가중점수(S1=5·S2=2·S3=0.5, 0~100): NN
등급: A|B|C|D
-->
```

## 5. 품질 등급

- **A:** S1 잔여 0, S2 ≤2, 학술 규범 훼손 0, 개선 70%+.
- **B:** S1 잔여 0, S2 ≤4, 규범 훼손 0, 개선 50%+.
- **C:** S1 잔여 1~2 또는 과교정 신호 또는 규범 1건 훼손 → 2차 패스.
- **D:** S1 잔여 3+ 또는 >50% 변경 또는 규범 다수 훼손 또는 인용 날조 의심 → 사람 검토 권고, 작업 중단.

## 6. 윤리 입장

이 도구의 목적은 **한국어 학술 글쓰기 품질 향상**이지 AI 탐지 회피·학술 부정이 아니다.
표절 검사·연구 진실성 검증 대체용이 아니며, **인용·데이터·주장은 저자가 책임진다.**
근거 없는 내용은 만들지 않고 플래그만 한다. 라이선스 Apache-2.0.
