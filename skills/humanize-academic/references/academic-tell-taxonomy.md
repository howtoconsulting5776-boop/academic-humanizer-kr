# 학술 AI-티 분류 체계 (Academic AI-Tell Taxonomy) v0.1

한국어 학술 텍스트(논문·학위논문·초록·학술지 원고)에서 LLM이 남기는 'AI 티'를 11개
범주로 정리한 단일 진실 원천(SSOT). 범용 휴머나이저(im-not-ai)의 A~J 체계를 **학술
레지스터에 맞게 재조정**했다. 핵심 차이: 헤징·명사화·피동·형식구조는 학술 규범이므로
**양면(two-sided) 범주**로 다루어 *슬롭만* 제거하고 *규범은 보존*한다.

## 심각도
- **S1:** 한 번만 나와도 AI/비학술 확정 → 처리(교정 또는 플래그).
- **S2:** 3회 이상 누적되면 티 남 → 밀도 기반 처리.
- **S3:** 다른 패턴과 중첩될 때 강화 → 리듬·구조 조정 수준.
- **KEEP:** 정당한 학술 규범 → `--academic` 모드에서 **절대 건드리지 않음**(보존 화이트리스트).

## 보존 제외(불가침) 토큰
고유명사, 숫자·통계·연도·p값·효과크기, 직접인용("..."), 참고문헌·인용키(저자, 연도),
수식·기호, 분야 표준 약어·전문용어. 이들이 포함된 span은 의미 변경 금지.

## 1. 공허한 메타·상투구 (Empty meta & boilerplate) — S1

인간 학자가 거의 쓰지 않는, 내용 없는 '목청 가다듬기' 문장. AI 학술 텍스트의 결정적
시그니처(MDPI 2025; ResearchLeap VERMILLION).

| 패턴 | before | after |
|------|--------|-------|
| 1-1 메타 서두 | "주목할 만한 점은 ~라는 것이다." / "It is important to note that ~" | (삭제하고 본론 직진) "~다." |
| 1-2 공허한 의의 | "이는 시사하는 바가 크다." / "중요한 의미를 가진다." | (구체화) "이는 X가 Y에 직접 영향을 줌을 뜻한다." 또는 삭제 |
| 1-3 결론 상투구 | "결론적으로, 요약하면, 앞서 살펴본 바와 같이" 남발 | 1회 이하로 제한, 대개 삭제 |
| 1-4 연구 목적 공식 남발 | "본 연구는 ~을 살펴보고자 한다"가 단락마다 반복 | 첫 등장만 두고 변주 |
| 1-5 자기지시 메타 | "본 절에서는 ~을 다룬다 / 다음으로 ~을 논한다" 고정 반복 | 절반 이상 삭제, 내용으로 직진 |

처방: 이 범주는 **삭제 우선**. 내용이 있으면 그 내용만 남긴다.

---

## 2. 근거 없는 일반화·단정 (Unsupported generalization) — S1 → 플래그

학술 글의 가장 치명적 AI 결함. 출처 없이 합의를 가장하거나("X로 알려져 있다") 인용 없이
규모를 단정한다("많은 연구가 ~을 보여준다"). 검색 결과: AI는 논쟁적 주제를 합의로
뭉뚱그리고 "X have been described as Y"를 누가·왜 없이 쓴다.

| 패턴 | before | 처리 |
|------|--------|------|
| 2-1 무출처 합의 | "~로 널리 알려져 있다 / 일반적으로 받아들여진다" | `[저자 확인: 출처 필요]` 플래그 + 단정 약화 |
| 2-2 무인용 규모 | "많은/다수의 연구들이 ~을 입증했다" (인용 0) | 플래그 또는 "선행 연구(예: )에서 ~" 빈칸 표시 |
| 2-3 논쟁 평탄화 | 대립 견해를 단일 합의처럼 서술 | 플래그 + "한편 ~라는 견해도 있다" 여지 |

**처방 원칙:** 절대 출처를 지어내지 않는다. **교정이 아니라 플래그**(주석)로 저자에게
넘긴다. 단정 어조만 학술 헤징으로 낮출 수 있다.

---

## 3. 환각·모호 인용 (Hallucinated / vague citation) — S1 → 플래그만

| 패턴 | 신호 | 처리 |
|------|------|------|
| 3-1 그럴듯한 가짜 인용 | 검증 불가한 (저자, 연도), 둥근 숫자 통계 | `[저자 확인: 인용 검증 필요]` — **수정·삭제·생성 모두 금지** |
| 3-2 모호 인용 | "한 연구에 따르면" (누구?) | 플래그, 본문 보존 |

이 범주는 **읽기 전용 플래그**다. 휴머나이저가 인용을 만들거나 고치면 안 된다.

## 4. Hedging 오용 — 양면 (4a 교정 S2 / 4b 보존 KEEP)

학술 헤징은 규범이다(Manchester Phrasebank; GMU Writing Center: 과잉단정은 신뢰를
깎는다). 따라서 **헤징 자체를 제거하지 않는다.** AI 티는 *중첩·공허한* 헤징이다.

### 4a 과잉·중첩 헤징 (교정, S2)
| before | after |
|--------|-------|
| "~할 수 있을 것으로 보일 수 있다" (양태 2~3중첩) | "~할 수 있다" 또는 "~로 보인다" (1중) |
| "아마도 ~일 가능성이 있다고 추정될 수 있다" | "~일 가능성이 있다" |
| "어느 정도, 다소, 비교적"이 한 문장에 다발 | 1개만 |

### 4b 정당한 단일 헤징 (보존, KEEP)
"~로 보인다 / ~일 수 있다 / ~할 가능성이 있다 / 본 연구의 결과는 ~을 시사한다 /
제한된 자료에 근거하면" — **단일 층위는 학술 규범. 절대 단정으로 바꾸지 않는다.**
(`--general` 모드에서만 단정화 허용.)

판별: 한 절에 양태·추측 표지가 **2개 이상 중첩**되면 4a(교정), 1개면 4b(보존).

---

## 5. 명사화·추상화 — 양면 (5a 해체 S2 / 5b 보존 KEEP)

학술 글에서 명사화는 정상이다(분야 표준 용어). AI 티는 *행위자를 은폐하는 연쇄
명사화*다(ResearchLeap: "cascades of abstract nouns that mask agency").

### 5a 행위자 은폐 연쇄 (해체, S2)
| before | after |
|--------|-------|
| "데이터의 분석을 통한 결과의 도출의 수행" | "데이터를 분석하여 결과를 도출했다" |
| "~성의 ~화를 통한 ~적 접근" (‑성/‑화/‑적 체인) | 동사구로 풀기: "~을 ~하게 만들어" |
| "carried out an exploration of" 직역식 "탐색을 수행하였다" | "탐색하였다" (Editage tip1) |
| 한 문장에 명사화 접미사(‑성·‑화·‑적·‑tion) 12회 초과 | 일부 동사·형용사 환원 |

### 5b 분야 표준 명사화 (보존, KEEP)
"타당성, 신뢰도, 정규화, 회귀, 상호작용효과, 매개효과" 등 **확립된 학술 용어는 보존.**
동사로 풀면 오히려 비학술적이 된다.

판별: 의미 손실 없이 동사로 풀 수 있고 행위자가 살아나면 5a, 표준 술어면 5b.

## 6. 번역투 (Translationese) — S1/S2

한국어 특화. 학술 글에도 그대로 적용(im-not-ai A 범주 계승). 경희대 글쓰기센터·
한국 번역학계 근거.

| 패턴 | before | after | 심각도 |
|------|--------|-------|--------|
| 6-1 ~를 통해 남발 | "분석을 통해 도출했다" | "분석하여 도출했다" | S2 |
| 6-2 ~에 있어서/~에 대하여 | "교육에 있어서 / X에 대하여 다룬다" | "교육에서 / X를 다룬다" | S1 |
| 6-3 이중 피동 | "측정되어진다 / 분석되어졌다" | "측정된다 / 분석되었다" | S1 |
| 6-4 가지고 있다 직역 | "유의성을 가지고 있다" | "유의하다" | S2 |
| 6-5 대명사 직역 | "그것은 / 그것들은" 영어 it 직역 | 생략 또는 명사 재사용 | S2 |
| 6-6 관계절 좌향수식 | "~한 ~한 ~인 N" 긴 좌향 수식 | 문장 분리 또는 동격절 | S2 |
| 6-7 이중조사 | "~에서의 / ~에의 / ~으로의" | 절·구로 풀기 | S2 |
| 6-8 ~라고 알려진 | "X라고 알려진 Y" | "X인 Y" / 병기 간소화 | S2 |

## 7. 리듬·구조 균일성 (Rhythm / structure uniformity) — S2/S3

AI는 균일 길이 문장과 고정 단락 형태를 만든다(VERMILLION: "uniform clause length,
paragraph blocks consistently 3–5 lines"). 학술 글이라도 인간 전문가는 리듬을 변주한다.

| 패턴 | 신호 | 처방 |
|------|------|------|
| 7-1 문장 길이 단조 | 대부분 30~50자, 표준편차 낮음 | 단문(10~20자)·장문(80자+) 혼용, stdev↑ |
| 7-2 동일 종결 반복 | "~다."만 연속 | "~다 / ~었다 / ~는다 / 명사형" 섞기(평서체 유지) |
| 7-3 단락 형태 고정 | 모든 단락 3~5문장 | 일부 단락을 길거나 1~2문장으로 |
| 7-4 동일 문두 반복 | "본 연구는~ / 본 절에서는~" 반복 시작 | 시작부 변주 |
| 7-5 ~고 있다 남발 | 진행상 과다 | 단순 시제로 |

학술 평서체(‑다)는 유지하되 **호흡만** 다양화. (경어법 혼재 금지 — 일관 유지.)

## 8. 접속사·전이 공식 (Connector / transition formulae) — S2

VERMILLION: "Moreover, Furthermore, In conclusion, Additionally" 남발 = 결정적 티.
한국어: "또한·게다가·나아가·따라서·즉" 문두 남발.

| 패턴 | 처방 |
|------|------|
| 8-1 문두 접속사 과다 | "또한·게다가·나아가" 70% 제거, 자연 연결로 |
| 8-2 고정 간격 전이 | 단락마다 같은 위치 전이어 → 위치·종류 변주 |
| 8-3 "즉" 남발 | 2회 이하 |
| 8-4 "이는 ~" 메타 진입 | 삭제하고 서술 직진 |

학술 논증의 정당한 논리 접속(따라서·그러므로 1~2회)은 보존.

---

## 9. Hype·주관 형용사 (Hype & subjective adjectives) — S1

학술 객관성 위반. Editage tip4: "interesting·remarkable 같은 주관 형용사는 피하고
판단은 독자에게." AI는 과장 어휘를 즐긴다.

| before | after |
|--------|-------|
| "혁신적·획기적·압도적·놀랍게도·흥미롭게도" | 삭제 또는 구체 근거로 대체 |
| "매우 중요한 / 대단히 유의미한" | "유의한(p<.05)" 등 구체 수치 |
| "주목할 만한 성과를 보였다" | "X% 향상되었다" |

처방: 평가어를 **증거**로 치환. 증거 없으면 삭제.

| 패턴 | 처방 |
|------|------|
| 10-1 em-dash(—) 남용 | 쉼표·괄호·문장 분리로 (VERMILLION: AI 시그니처) |
| 10-2 과도한 **볼드** | 학술 본문에서 거의 제거 |

| 10-4 산문 속 불릿·번호 남발 | 산문화 (단, 정당한 목록·표는 보존) |
| 10-5 따옴표 과다 | 강조용 따옴표 최소화(직접인용은 보존) |

## 11. 피동 남용 — 양면 (11a 일부 능동화 S2 / 11b 보존 KEEP)

Editage tip4: 학술 글은 객관성 위해 피동을 **선호**한다. 따라서 피동을 일률 제거하지
않는다. AI 티는 *모든 문장*이 기계적 피동일 때다.

### 11a 기계적 피동 일변도 (일부 능동화, S2)
모든 문장이 "~되었다 / ~된다"로 끝나 단조로우면 일부를 능동·연구자 주어로 환기:
"분석이 수행되었다" → "본 연구는 ~을 분석하였다"(분야 관행 허용 시).

### 11b 객관성 피동 (보존, KEEP)
방법·결과 서술의 "측정되었다 / 통제되었다 / 표집되었다"는 학술 규범 → 보존.
`--general`에서만 적극 능동화.

## 검출·윤문 계약

탐지기 출력 JSON 스키마:
```json
{
  "findings": [
    {
      "category": "1~11",
      "severity": "S1|S2|S3",
      "text_span": "원문 그대로",
      "suggested_fix": "교정안 또는 null(플래그만)",
      "is_flag_only": true,
      "rationale": "근거 한 줄",
      "keep_norm": false
    }
  ]
}
```
- `is_flag_only: true` → 범주 2·3. 본문 수정 금지, 주석 플래그만.
- `keep_norm: true` → 4b·5b·11b 후보. `--academic`에서 처리 제외.

**가중 심각도:** S1=5, S2=2, S3=0.5 → 0~100 정규화. 플래그는 점수에서 분리 집계.
